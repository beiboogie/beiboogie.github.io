## 效果

在制作博客时，需要加入图片帮助阐述。通过使用`Image Converter for ObsidianMD`插件，可以在“粘贴图片”这个动作时自动完成：

- 压缩图片
- 转换为webp格式
- 自动将图片放入目标文件夹
- 自动填写正确的引用

基本上也就是写博客需要的所有图片操作步骤了。非常方便

## 功能

支持的图片格式：WEBP、JPG、PNG、HEIC、TIF

- 🖼️ **转换**：自动将拖放/粘贴的图片转换为 WEBP、JPG 或 PNG
    
- 🗜️ **压缩**：通过指定 1-100 之间的质量值来减小文件大小
    
- 📏 **调整**图片大小（有损和无损）
    
    - 自动读取图片尺寸并将其应用到图片链接，例如：将图片宽度应用到 `|width` 或指定任何自定义尺寸。
    - 通过拖动图片边缘或使用滚轮（例如，**CMD+滚轮**）来调整大小
    - 调整**原始**图片大小（宽度、高度、最长边、最短边、适应、填充）
- 对齐（左、右、居中）并在图片周围环绕文本，无需在链接中使用任何自定义语法：
    
- **图片标注**和**标记工具**。直接在 Obsidian 中在图片顶部绘制、书写、涂鸦、注释、标记。
    
- ✂️ **裁剪、旋转和翻转图片**
    
- 📁 **自定义文件管理和重命名**：
    
    - **重命名**：使用变量（例如，`{noteName}`、`{fileName}`）自动重命名图片[支持的变量列表](docs/Variables%20Reference%20Guide.md)
    - **输出**：使用变量将图片组织到自定义输出文件夹中。[支持的变量列表](docs/Variables%20Reference%20Guide.md)
- 🌍 **纯 JavaScript 实现**，完全**离线**工作。无需外部 API 或二进制依赖（例如 ImageMagick、Cloudinary、FFmpeg、sharp 等）——保持轻量、便携和安全。
    

## 其他

- 🔄 **批量处理**：转换、压缩笔记或整个仓库中的所有图片。
- 🔗 **与其他 Markdown 编辑器的兼容性**：能够为图片使用 **Markdown 链接**，为所有其他链接使用 **Wiki 链接**。
- 🖱️**自定义右键上下文菜单：**
    - 复制到剪贴板
    - 复制为 Base64 编码的图片
    - **调整**刚刚点击的原始图片大小
    - **从仓库中删除图片** - 从仓库中移除图片及其链接
