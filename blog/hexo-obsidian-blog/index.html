最终效果

按照下文的方法配置后，完整的写作工作流可以简化为：

  1. 打开Obsidian
  2. 进入_posts文件夹
  3. 快捷键 - 新建文章并填写标题
  4. 快捷键 - 应用模板并填写属性
  5. 写作，利用Attachment Management插件实现 copy & paste 图片嵌入
  6. 执行hexo clean && hexo d 部署

准备和思路

将你的Hexo根目录添加为Obsidian仓库。如果你使用 Hexo提供的一键部署，你将不需要做任何其他调整，不需要修改.gitignore。

总体上要做的事情就是：
用Obsidian强大的自定义功能去“迎合”Hexo的博客框架，充分利用Hexo和Obsidian两者各自的方便之处。这也是为什么没有使用Obsidian的git插件，因为Hexo的一键部署已经很好用。

这样修改后，Hexo会在你每次执行hexo new 你的文章名指令的时候在_posts文件夹下创建新的文章和与文章同名的文件夹。（这个过程也可以你手动完成，是一样的。）

标准的格式下，你应当在这个同名文件夹下防止你文章需要的图片，并在文章中用![图片描述](图片名.png)来插入图片。如果你使用过typora或者obsidian，你应该会知道这和这两个软件的图片插入逻辑有所区别。因此我们需要通过插件来迎合Hexo的框架。

网络上很多人推荐的Custom Attachment Location插件在我的使用环境下会内存泄漏，原因未知。但机缘巧合，我发现了一个更好的插件：Attachment Management。安装后启用，它的默认配置就是刚好符合上面所说的Hexo的框架的。因此你可以在任意地方复制图片并Ctrl+V到文章对应的地方，插件会自动为图片重命名为你可以设置的格式并存储在同名文件夹下。引用的格式也是正确的。

感想

如你所见，我希望搭建一个尽可能简单的博客。
